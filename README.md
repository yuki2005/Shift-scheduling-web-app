# Restaurant Shift Scheduling Web Application
---

## 1. 概要（Overview）

本アプリケーションは、私が学生時代にアルバイトをしていた飲食店をモデルとして開発した
**シフト自動生成Webアプリケーション**である。

シフト作成における人的負担や作業時間の削減を目的とし、
従業員のスキル・希望シフト・日ごとの必要人数を考慮した
自動シフト生成ロジックを実装した。

生成結果はデータベースに保存され、
履歴の参照、内容の確認、時間帯・ポジション単位での手動修正が可能となっている。

従業員の希望、スキル、時間帯ごとの必要人数を考慮しながら  
シフトを自動生成し、保存・閲覧・手動修正まで行うことができます。

## 2. 主な機能（Features）

- 従業員情報の管理
- 希望シフトの入力・管理
- シフトの自動生成

- 保存済みシフトの一覧表示、日付での検索
- シフトの手動修正（時間帯×ポジション単位）
- 重複割り当ての検出・警告表示

## 8. シフト自動生成アルゴリズム

- 従業員の希望可否
- スキル
- 時間帯ごとの必要人数
- ポジションの重要性

これらを考慮して  
**スタッフ選定戦略（Strategy）**として実装しています。

実際の現場では、体調や人間関係、突発的な事情など、
数値化・モデル化できない要素が多く存在します。
そのため、自動で「完璧なシフト」を生成することは現実的ではないと考え、
本アプリケーションでは厳密な最適化問題として定式化することは行いませんでした。

代わりに、制約を満たすことを最優先とし、
業務上重要なポジションから順に割り当てを確定させる
貪欲法によるヒューリスティックな手法を採用しています。

## 9. 実装上の工夫点

### 9.1 Entity と DTO の分離

- Entity をそのまま API レスポンスに使わない
- 循環参照（無限JSON生成）の回避
- フロントエンドに不要な情報を送らない

### 9.2 確定シフト保存設計

- `finalAssignmentJson` に JSON 形式で全体を保存
- 明細テーブルで編集・検索可能に

## 10. 発生した主なエラーと解決

### JSON パースエラー
- 原因：JSON構造の不整合
- 対応：バックエンド・フロント双方で形式を統一

### Hibernate TransientPropertyValueException
- 原因：親Entity未保存状態で子Entityを保存
- 対応：保存順序の修正（Record → Assignment）

これらを通して  
**エラーログを読んで原因を特定する力**を身につけました。

## 11. 開発環境・ビルド設定に関する問題と解決

### 背景

開発初期段階では、Java のパッケージを src フォルダ直下に配置して開発を行っていました。
その後、Maven / Spring Boot の標準構成に合わせるため、
Java ソースを src/main/java 配下へ移動する対応を行いました。

### 発生した問題

- package 宣言とディレクトリ構成の不一致による大量のエラー表示
- Source Folder の設定が正しく認識されない
- IDE 上ではエラーが表示されるが、アプリケーション自体は起動・動作する状態
- Maven Update を行うたびに Build Path が変化し、エラーが再発

### 原因

IDE の設定だけでなく、pom.xml を確認したところ、
以下の設定が原因であることが分かりました。

```xml
<sourceDirectory>src</sourceDirectory>
```



この設定により、Maven が
src/main/java ではなく src 全体を Java ソースディレクトリとして解釈しており、
Eclipse（m2e）がプロジェクト更新時に Build Path を自動的に再構成することで、
Source Folder の不整合が発生していました。

### 対応

- pom.xml から `<sourceDirectory>` 設定を削除
- Java ソースを `src/main/java` 配下に統一
- Maven Update Project を実行し、Build Path を再生成

### 学び

- Maven プロジェクトでは **標準ディレクトリ構成を前提に設計することが重要**
- IDE 上のエラー表示と、実行時の挙動は必ずしも一致しない
- 問題の原因は IDE 設定だけでなく、`pom.xml` にある場合がある

この経験を通して、
ビルドツール・IDE・実行環境の関係を意識して原因を切り分ける必要があることを学びました。
